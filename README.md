# AI Company — 仮想カンパニーで業務を組織的に回すClaude Codeプラグイン

Claude Code上に仮想組織を構築し、秘書が窓口となって業務を振り分け・管理するプラグインです。
[cc-company](https://github.com/Shin-sibainu/cc-company)（Shin-sibainu氏、MIT License）をベースに、実運用ノウハウを追加したカスタム版です。

## 特徴（cc-companyからの追加要素）

### レビュー部門
- ファクトチェック専門のシニアアナリストがレビューを担当
- A-D評価フォーマットで品質を可視化
- 改善提案を必ずセットで出力

### 4要素フレームワーク
部門CLAUDE.mdを以下の4要素で構成:
- **ペルソナ**: 専門家としての人格定義
- **絶対原則**: 破ってはならないルール
- **能動的アクション**: 指示がなくても実行すべきこと
- **過去の教訓**: 運用で得た知見の蓄積

### スナップショット自動保存
- セッション終了時に全部署の状態を自動保存
- 次回セッション開始時に最新スナップショットを読み込み
- セッション間のコンテキスト維持を実現

### リサーチ部門の4原則
1. **WebSearch必須**: 推測に頼らず必ず検索する
2. **出典付与**: URL + 取得日を必ず記載
3. **未確認正直**: 確認できなかった情報は正直に「未確認」とする
4. **一次資料優先**: 二次情報より公式ソース・一次資料を優先

### 業種別テンプレート
- 士業（弁護士・税理士・行政書士など）
- EC・小売
- 飲食・店舗
- 不動産

「業種テンプレートを使う」で最適な部門構成を一括セットアップ。

### ダッシュボード自動生成
- 全部署の概要をKPI付きで表示
- リサーチ本数、レビュー通過率、TODO完了率を集計
- HTML出力オプションで可視化

### 週次レビュー自動実行
- 「週次レビュー」コマンドで全部署の成果を集計
- 来週のTODO提案を自動生成
- `secretary/notes/YYYY-MM-DD-weekly-review.md` に保存

### 部門間連携ログ
- リサーチからレビューなど部門間受け渡しを自動記録
- `secretary/notes/YYYY-MM-DD-handoff-log.md` に保存

### KPIトラッキング
- 部署ごとのKPIをダッシュボードで一元管理
- 期間指定で推移を確認可能

### アラート機能
- デッドライン接近（TODO期限の前日に警告）
- リサーチの鮮度切れ（30日以上経過したデータにフラグ）
- 未処理inbox（5件以上溜まったら通知）
- セッション開始時に自動チェック

### エクスポート機能
- **Notion**: Markdownのままコピペ可能な形式で出力
- **Obsidian**: `[[内部リンク]]` 形式でファイル間リンクを自動生成

### 実績レポート生成
- 指定期間の成果をまとめたレポートを自動生成
- リサーチ本数、動画制作数、意思決定数を自動集計
- note記事やコンサル報告書として使える形式で出力

## インストール

Claude Codeを起動した状態で、以下の2ステップを実行してください。

**Step 1: マーケットプレイスを追加**
```
/plugin marketplace add pontarou-re/ai-company
```

**Step 2: プラグインをインストール**
```
/plugin install company@ai-company
```

## クイックスタート

```
/company
```

秘書がオンボーディングを開始します。2つの質問に答えるだけで、仮想組織が立ち上がります。

## コマンド例

| コマンド | 説明 |
|---------|------|
| `/company` | 運営モード開始 |
| 「ダッシュボード」 | 全部署の概要+KPIを表示 |
| 「週次レビュー」 | 今週の成果を集計 |
| 「実績レポートを作成」 | 期間指定で成果レポート生成 |
| 「業種テンプレートを使う」 | 業種別の部門構成を一括セットアップ |
| 「Notionにエクスポート」 | Notion向け形式で出力 |
| 「Obsidianにエクスポート」 | Obsidian向け形式で出力 |
